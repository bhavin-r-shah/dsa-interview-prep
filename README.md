# Approach to writing a program

## Open Questions
- Ask questions to clarify my understanding of the question
- Ask about corner cases
- Ask about error handling
- Ask about posible inputs and possible outputs

## Assumptions:
- Clearly state assumptions
- Helps clarify scope of the problem

## Fact:
- Clarify facts
- E.g. In a Fibonaci series each number is the sum of the previous 2 numbers in the series, starting from 0 and 1.

## Error Handling
- Validate inputs and send required errors.
- Any exceptions to handle? e.g. File_Not_Found, Array_Index_Out_of_Bounds
- Make sure program exits gracefully in case of any errors

## Pseudo Code:
- Write a psuedo code to get mental map of the solution

## Time & Space Complexity
- Important to consider these in order to determine if there are better / neater ways to handle this problem
